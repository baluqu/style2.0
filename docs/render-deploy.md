# Deploy StyleBridge on Render (Free Tier)

This project is now set up for a straightforward Render free web service using the checked-in [`render.yaml`](../render.yaml).
For an execution checklist, use [`docs/72-hour-sprint.md`](./72-hour-sprint.md).

## 1) Prep the repo safely

1. Make sure `.env` is not committed.
2. Keep secrets only in Render environment variables.
3. Generate strong keys for production:
   - `SECRET_KEY`
   - `ADMIN_KEY` (if admin grant flow is used)

## 2) Push to GitHub

1. Create a public GitHub repo.
2. Add it as `origin`.
3. Push the current branch.

## 3) Deploy on Render

1. In Render: `New` -> `Blueprint`.
2. Connect your GitHub repo.
3. Render reads `render.yaml` and creates a free Python web service.
4. Wait for deploy; Render should pass the health check at `/health`.

## 4) Required environment values

These are already declared in `render.yaml`:

- `FLASK_ENV=production`
- `LOG_LEVEL=INFO`
- `SECRET_KEY` (generated by Render)
- `ADMIN_KEY` (generated by Render)
- secure cookie + proxy flags

## 5) Database choice

- MVP default: SQLite (already works out of the box).
- Scale later: add free Postgres and set `DATABASE_URL`.

## 6) Upload note

Guest uploads are temporary on free hosting. For durable uploads, switch to object storage (for example Supabase Storage or S3-compatible storage) and set the related env vars.
